# zzDiary 智能日记本 — 软件设计文档

## 1. 产品概述

**zzDiary** 是一款以正念（Mindfulness）为核心的智能日记分析工具。它通过对话式书写体验，引导用户逐步表达内心难以启齿的想法与情绪，并结合原生家庭回溯，帮助用户理解情绪根源，推进自我觉察。

## 2. 核心功能

### 2.1 引导式日记书写

- AI 以温和、非评判的语气逐轮引导，降低表达门槛。
- 针对用户回避的话题，采用渐进式提问（"不想写也没关系，可以简单说说今天的感受"）。
- 支持自由书写模式与引导模式一键切换。

### 2.2 情绪分析与洞察

- 每篇日记自动生成情绪标签（如焦虑、愤怒、羞耻、喜悦）。
- 输出情绪趋势图（周/月/年），标记情绪拐点。
- 关键句高亮，标注可能的认知偏差（如灾难化、读心术）。

### 2.3 原生家庭回溯

- 在用户允许下，AI 会询问家庭背景（父母关系、童年经历、教养方式）。
- 将当前情绪/行为模式与原生家庭经验关联分析，生成"情绪来源推测"。
- 所有家庭信息仅在本地存储，不上传云端。

### 2.4 正念练习引导

- 书写后推荐匹配当前情绪的正念练习（呼吸练习、身体扫描、慈心冥想）。
- 每日"感恩三件事"轻量模板，培养积极关注习惯。

## 4. 技术选型

| 层次 | 技术 | 说明 |
|------|------|------|
| 前端框架 | Tauri + React | 轻量桌面应用,打包体积小 |
| 本地数据库 | SQLite | 日记、情绪标签、家庭背景数据 |
| 向量检索 | lanceDB (本地) | 历史日记语义搜索 |

| 加密 | SQLCipher | 日记内容加密存储 |
| 图表 | ECharts / Recharts | 情绪趋势可视化 |

## 6. AI 引导策略

### 6.1 对话框架

1. **开场** — "今天有什么想聊的吗？"
2. **深化** — 若用户情绪词密度高，追问："这件事让你联想到过去的类似经历吗？"
3. **回溯** — 若已录入家庭背景："你刚才说的这种被忽视的感觉，和你之前提到小时候和妈妈的关系有联系吗？"
4. **收束** — 生成当日小结 + 正念建议。

### 6.2 Prompt 设计原则

- 始终以用户主体性为优先，避免强行分析。
- 使用"可能"、"或许"等开放性措辞，避免诊断式语气。
- 对敏感话题自动切换更温和的表达。

## 7. 隐私与安全

- 所有日记与家庭数据仅存储在本地，默认不上传任何服务器。
- 若使用云端 API（如 DeepSeek），发送前做脱敏处理，且明确告知用户。
- 支持 Ollama 本地模型部署，实现完全离线使用。
- 应用启动可设密码/生物识别锁。

## 9. 模型调用方案

首选 **DeepSeek Chat API**（`deepseek-chat` 模型），原因：
- 中文理解能力强，适合日记分析场景。
- 免费额度充足，API 价格极低。
- 支持长上下文，便于引入历史日记与家庭背景。

备选方案：通过 **Ollama** 本地运行 `qwen2.5` 或 `deepseek-r1` 蒸馏版，实现离线分析。
